# No-Cost Learning Rules

QF-101 is designed to run locally at no cost.

## Cost Rules For QF-101

- QF-101 requires no paid services.
- Do not enter credit-card details for QF-101.
- Do not create AWS resources for QF-101.
- Do not use Amazon Braket managed notebooks or paid QPUs for QF-101.
- Do not use IBM Pay-As-You-Go for QF-101.
- IBM Quantum hardware access may have free limited access, but QF-101 does not require hardware.
- Cloud or hardware execution is reserved for later advanced optional labs with explicit warnings.

## Account And Secret Safety

Never paste API keys, tokens, secrets, private account details, private file paths, or private credentials into:

- public GitHub Issues
- lab submissions
- learner check-ins
- code files
- screenshots
- report templates

If a tutorial, video, website, or tool asks for payment details or cloud credentials while you are doing QF-101, stop and return to the local setup guide.

## What QF-101 Uses Instead After Enrollment

QF-101 is designed around:

- public course overviews
- enrolled course materials
- local simulation
- public-safe GitHub Issues for help
- manual review for public learning milestones

Use [Lab help](https://github.com/Goblin-Quantum-Lab/goblin-quantum-lab/issues/new?template=lab_help.yml) if you are unsure whether a setup step is safe.
