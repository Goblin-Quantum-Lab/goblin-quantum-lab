# Module 4: Observing Results

## Purpose

This module teaches learners how to read simple result counts and describe what those results support without overstating what they prove.

## Result Counts

A quantum experiment often reports counts after repeated shots.

Example format:

| Outcome | Count |
| --- | ---: |
| `0` | 48 |
| `1` | 52 |

Counts are observations. They are not automatically the same thing as the exact underlying probabilities.

## Histograms

A histogram is a visual way to show counts. Taller bars represent outcomes that appeared more often.

For beginner work, a table is enough if no plotting tool is available. The important skill is reading the pattern and explaining it carefully.

## Expected Result

The expected result is what you predict before running or inspecting the experiment. In disciplined lab work, write this expectation before looking at the final counts.

An expectation might describe:

- Which outcome should appear most often
- Whether the result should look balanced
- How much variation might be reasonable for a small shot count

## Observed Result

The observed result is what the experiment actually reports. It may not match the expectation perfectly, especially with small shot counts.

Careful learners compare expected and observed results without forcing the data to look cleaner than it is.

## Sampling Variation

Sampling variation means that repeated samples can look different. If you run the same probabilistic experiment several times, the counts may shift.

Small samples can swing more noticeably. Larger shot counts usually give a clearer view of the distribution, but they still do not remove every source of uncertainty.

## Careful Interpretation

A careful interpretation should say what the result supports and what it does not prove.

Useful sentence starters:

- "The result is consistent with..."
- "The counts suggest..."
- "This does not prove..."
- "A limitation is..."

Avoid overstatements such as:

- "This proves the qubit was both values."
- "This proves the simulator is perfect."
- "This proves quantum computers are always faster."

## Mini Task

Given the result table below, explain what the result supports and what it does not prove.

| Outcome | Count |
| --- | ---: |
| `0` | 6 |
| `1` | 4 |

Write two or three careful sentences. Do not include a solution guide.

## Vocabulary

- **Expected result**: The result pattern predicted before observing the data.
- **Observed result**: The result pattern actually recorded.
- **Sampling variation**: Natural variation caused by finite samples.
- **Interpretation**: A careful explanation of what the result suggests.
