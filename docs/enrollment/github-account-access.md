# GitHub Account Access

Goblin Quantum Lab uses GitHub-native enrollment for enrollment-required courses. Support Subjects remain public and can be read without a site account.

## Why A GitHub Account Is Required

Enrollment-required courses need a stable learner identity so maintainers can connect requests, submissions, review comments, badge requests, and course access decisions to the same person. For now, that identity is the learner's own GitHub account.

Learners must use their own GitHub account. Do not request enrollment using someone else's account.

## Access Rules

- Enrollment is manually reviewed.
- Access is not instant.
- Course access is account-bound.
- Assessed work is GitHub-account-bound.
- Course access may be refused, delayed, paused, or revoked if policy is violated.
- Advanced work may be prerequisite-gated.

## Public And Enrolled Content

- Support Subjects remain public.
- Public course pages are overviews only.
- Assessed work is handled through the enrolled learner process.
- Full course materials, course slides, course PDFs, lab files, assignments, tests, submission instructions, and learner-specific review instructions are enrolled-only.

Protected course materials and assessment workflows are maintained outside the public repository.

## Learner Responsibilities

Learners must not share protected course materials. Learners must not publish assignments, tests, protected responses, reviewer feedback, badge decision records, private process details, or enrolled-only instructions.

GitHub Issues are public. Do not submit passwords, tokens, API keys, government ID numbers, private financial information, sensitive personal information, protected assessment materials, or reviewer notes.
